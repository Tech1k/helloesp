    SD Card Backup & Restore
    
        
    
            
            
    
            
    
            
    
            
            
            
    
            
    
            
    
            
    
            
    Off-site backup (R2)
    
            
    
                Fires daily at 4 AM local. Stores the full SD card under state/YYYY-MM-DD/ on Cloudflare R2, excluding config.txt, *.tmp/*.bak, and /logs. Rotation: 7 daily + 4 weekly + 12 monthly + yearly forever. Falls back to email attachment if R2 isn't bound.

                How to restore from R2
                
    
                    If the SD card fails and you need to rebuild:
                    
      
                        
    1. Format a new FAT32 SD card (≤32 GB)
      2. 
                        
    2. Copy the data/ folder from the git repo to the SD root (HTML, images, etc.)
      3. 
                        
    3. Open the Cloudflare R2 dashboard, find the helloesp-backup bucket
      4. 
                        
    4. Read state/latest.json to see the most recent snapshot date
      5. 
                        
    5. Download every object under state/{date}/ and place it on the SD card, preserving paths (e.g. stats/weekly/2026-W16.json)
      6. 
                        
    6. Recreate config.txt from your own records (not backed up)
      7. 
                        
    7. Insert the SD card and boot
      8. 
                    
    
                    For partial fixes (just one corrupted file), download the single object from R2 and drop it through this admin panel's file upload.

    Chronicle Notes
    
        
    
            
    
                Add a curatorial note to any sealed Chronicle entry. Notes
                appear in italic above the auto-generated body on
                /chronicle,
                and post once as a reply to the entry's tweet on X. First
                non-empty save posts; later edits update the website only.
                Empty save clears the note (the X reply, if any, stays).
                Latest 3 entries shown by default; X reply engagement is
                strongest within hours of an entry's seal. Use the date
                jumper or "show older" toggle below for retroactive notes
                on older entries.
